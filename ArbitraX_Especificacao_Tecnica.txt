Sumário



1. Escopo Geral do Sistema
O ArbitraX é uma plataforma de arbitragem virtual com dois canais distintos: (1) WhatsApp Business (Meta Cloud API) para as partes do conflito — conversa individual 1-on-1 com o bot, sem grupos — e (2) painel web para árbitros, advogados e administradores. A IA auxilia na análise de provas e geração de projetos de sentença, sempre validados por árbitros humanos via painel web. Em conformidade com a Lei 9.307/96.

Por que Meta Cloud API e não Twilio?
Até 1.000 conversas/mês gratuitas (service conversations)
Custo por conversa muito menor: ~R$ 0,25–0,50 vs R$ 0,50–1,50 no Twilio
API nativa da Meta — sem intermediário, menor latência
Suporte a mídia (docs, fotos, vídeos, áudio) nativamente
Templates de mensagem aprovados para notificações (convite, prazos, sentença)
Economia estimada: R$ 2.000–5.000/mês em escala vs Twilio

2. Roles (Perfis de Usuário)
RBAC com perfis que refletem a estrutura processual. Diferença-chave: árbitro opera 100% no painel web; partes operam no WhatsApp.

2.1 Requerente (Autor)
Canal: WhatsApp (1-on-1) + Web (formulário e painel)
Iniciar pedido de arbitragem via formulário web
Pagar taxa de registro (web)
Assinar compromisso arbitral (web / Clicksign)
Enviar petição inicial e provas via WhatsApp (conversa com bot)
Enviar provas adicionais quando solicitado pelo bot
Receber notificações de prazo via WhatsApp
Receber sentença final via WhatsApp + download no painel
Acompanhar caso no painel web (visão opcional)

2.2 Requerido (Réu)
Canal: WhatsApp (1-on-1) + Web (aceitação e painel)
Receber convite de arbitragem via WhatsApp
Aceitar/recusar via link (página web)
Assinar compromisso arbitral (web / Clicksign)
Enviar contestação e provas via WhatsApp
Receber notificações e sentença via WhatsApp

2.3 Advogado
Canal: Painel Web (100%)
Acompanhar casos dos clientes no painel
Peticionar em nome do cliente via painel web
Upload de provas via painel
Gerenciar prazos de múltiplos clientes
Receber notificações por email

2.4 Árbitro
Canal: Painel Web (100%) — NÃO usa WhatsApp
Receber designação de caso no painel
Acessar peças, provas e timeline do caso no painel
Receber projeto de sentença da IA no painel
Aprovar sentença ou enviar sugestões de melhoria via painel
Ratificar sentença final via painel
Solicitar provas adicionais (o bot notifica as partes)
Declarar impedimento/suspeição
Consultar histórico de casos e honorários

2.5 Administrador
Canal: Painel Web (100%)
CRUD de usuários, árbitros, casos
Designar árbitros para casos
Configurar prompts e modelos da IA
Gerenciar templates (compromisso, sentença)
Dashboard financeiro: receita, honorários, inadimplência
Audit logs e métricas
Config. sistema: prazos, taxas, parâmetros

8. Fluxo WhatsApp Detalhado (1-on-1)
Cada parte conversa individualmente com o bot. Não existe grupo. O bot gerencia o estado de cada caso e direciona mensagens para a pessoa correta no momento certo.

8.1 Fluxo do Requerente
Requerente preenche formulário web → paga taxa → caso criado
Bot envia WhatsApp ao requerente: “Caso ARB-2026-00015 criado. Convite enviado ao requerido.”
Quando requerido aceita e ambos assinam: bot avisa requerente
Bot abre prazo: “Você tem 10 dias para enviar sua petição inicial e provas. Envie tudo aqui neste chat.”
Requerente envia textos e documentos no chat. Bot confirma cada recebimento.
Se IA solicita provas adicionais: bot pede ao requerente especificamente o que falta
Quando sentença é publicada: bot envia PDF + código de verificação

8.2 Fluxo do Requerido
Bot envia convite: “Você foi convidado para arbitragem. Caso: [resumo]. Clique para aceitar: [link]”
Requerido clica → página web → aceita → assina compromisso
Bot envia: “Compromisso assinado. Aguardando petição do requerente.”
Quando petição protocolada: bot avisa: “Você tem 10 dias para contestar. Envie aqui.”
Requerido envia contestação e provas no chat
Sentença publicada: bot envia PDF individualmente

8.3 Fluxo do Árbitro (Painel Web)
Admin designa árbitro → notificação no painel web (badge/alerta)
Árbitro acessa caso no painel: lê peças, provas, timeline
Quando IA gera projeto de sentença: notificação no painel (+ email opcional)
Árbitro abre projeto no painel. Lê ementa, relatório, fundamentação, dispositivo.
Opção A: clica “Aprovar” → sentença avança para ratificação
Opção B: escreve sugestões → IA gera nova versão → árbitro reavalia
Após aprovação: clica “Ratificar” na versão final
Sentença publicada. Honorários liberados.

9. Segurança e Compliance

9.1 Sigilo
Confidencialidade obrigatória (art. 13, Lei 9.307/96)
Conversas 1-on-1 (sem grupo) reforçam sigilo — partes não vêem msgs uma da outra
Criptografia at-rest AES-256 + in-transit TLS 1.3
RLS no Supabase: cada usuário só vê seus próprios casos
Audit log de todos os acessos

9.2 Auth
JWT access (15min) + refresh (7d)
RBAC por endpoint
2FA obrigatório para árbitro e admin
Rate limiting: 60 req/min

9.3 Integridade
Hash SHA-256 de cada documento
Assinatura Clicksign com validade jurídica
Código de verificação na sentença
Append-only: peças e provas não editáveis

9.4 LGPD
Consentimento explícito no compromisso
Direito de acesso e portabilidade
Retenção: 5 anos
DPO designado
