5.2 Models
model User {
  id String @id @default(uuid())
  nome String
  cpf_cnpj String @unique
  email String @unique
  telefone String @unique
  senha_hash String
  role UserRole
  oab_numero String?
  ativo Boolean @default(true)
  created_at DateTime @default(now())
  @@index([role, telefone])
}

model Arbitragem {
  id String @id @default(uuid())
  numero String @unique // ARB-YYYY-NNNNN
  requerente_id String
  requerido_id String?
  adv_requerente_id String?
  adv_requerido_id String?
  objeto String
  valor_causa Decimal @db.Decimal(12,2)
  categoria CategoriaArbitragem
  status ArbitragemStatus @default(AGUARDANDO_PAGAMENTO_REGISTRO)
  // SEM whatsapp_group_id — não há grupo
  created_at DateTime @default(now())
  compromisso Compromisso?
  pecas Peca[]
  provas Prova[]
  sentencas Sentenca[]
  prazos Prazo[]
  cobrancas Cobranca[]
  @@index([status, created_at])
}

model Sentenca {
  id String @id @default(uuid())
  arbitragem_id String
  versao Int @default(1)
  conteudo_texto String
  pdf_url String?
  hash_sha256 String?
  codigo_verif String? @unique
  status SentencaStatus @default(RASCUNHO)
  gerada_por String // ia | admin
  aprovacoes SentencaAprovacao[]
  @@unique([arbitragem_id, versao])
}

model SentencaAprovacao {
  id String @id @default(uuid())
  sentenca_id String
  arbitro_id String
  acao String // aprovar | sugerir | ratificar
  sugestoes_texto String?
  created_at DateTime @default(now())
  @@unique([sentenca_id, arbitro_id, acao])
}

model WhatsAppMessage {
  id String @id @default(uuid())
  arbitragem_id String
  user_id String
  wa_message_id String? @unique // Meta Cloud API message ID
  direcao String // inbound | outbound
  tipo String // text | document | image | video | audio
  conteudo String?
  media_url String?
  created_at DateTime @default(now())
  @@index([arbitragem_id, created_at])
}

6.2 Regras da IA
IA NUNCA decide — sempre gera PROJETO validado por árbitro humano (painel web)
Aplica: Lei 9.307/96, equidade, usos e costumes
Isolamento total entre casos (contexto separado)
Temp: 0.3 (análise), 0.5 (sentença). Max 5 versões.
Sentença: ementa + relatório + fundamentação + dispositivo + custas

6.3 Regras de Pagamento
Caso só avança após pagamento da etapa
Taxa arbitral split 50/50
Honorários liberados após sentença publicada
Reembolso: 100% antes instrução, 50% durante, 0% após sentença

6.4 Regras WhatsApp 1-on-1
Cada parte conversa individualmente com o bot — sem grupo
Partes não vêem mensagens uma da outra (sigilo reforçado)
Bot identifica contexto pela arbitragem_id vinculada ao telefone
Se parte tem múltiplos casos: bot pergunta qual caso antes de protocolar
Mídia recebida é catalogada automaticamente por tipo (doc, img, video, audio)
Mensagens armazenadas na tabela whatsapp_messages para audit trail
